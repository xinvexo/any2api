# ADR-0030: Gateway API Key 本地调用统计

- 状态：Accepted
- 日期：2026-07-23
- 决策者：maintainer

## 背景

管理面已经为每次完成的公开请求保存 `RequestLog`，其中包含认证使用的
`gateway_api_key_id` 和最终 HTTP 状态码。Gateway API Key 列表目前只能展示
`last_used_at`，管理员无法快速判断某把 Key 是否仍在使用、成功率如何以及最近
请求是否稳定。

## 决策

- 在 Gateway API Key 管理响应中返回按 `gateway_api_key_id` 聚合的本地观测：总请求数、成功数、失败数和最近 24 次最终请求状态码。
- 最终状态码为 2xx 计为成功，其余状态码计为失败；每个公开请求只计一次，`RequestAttempt` 不单独计入。
- 统计读取当前 RequestLog 保留窗口；日志关闭、无记录或统计暂时不可用时，配置接口仍成功，返回零值或空结果。
- 最近结果只返回状态码，不返回请求体、模型内容、认证材料或任何 Secret；React 负责把 2xx、4xx、其他状态分别显示为成功、警告和失败点。
- 统计只用于本地管理面观测，不参与 Gateway Key 权限、配额、计费、路由、熔断或重试。
- 通过按 `gateway_api_key_id`、时间和请求 ID 建立索引，聚合查询不改变 RequestLog 的持久化生命周期。

## 后果

- 管理员可以在 Key 列表中直接看到累计成功/失败和最近调用走势，无需打开请求日志页逐条筛选。
- 统计会随 RequestLog 保留策略删除旧数据；它不是永久计数器，也不代表计费口径。
- SQLite 聚合查询发生在管理读请求中，数据面不增加同步 I/O；遥测写入仍通过既有有界队列异步完成。

## 验证

- Storage 测试覆盖多 Key 聚合、2xx/非 2xx 分类、最近结果排序和无关联日志。
- Admin 契约测试覆盖列表与创建/轮换响应中的统计字段。
- React 契约和组件测试覆盖成功率、状态点阵、空统计与窄屏布局。
